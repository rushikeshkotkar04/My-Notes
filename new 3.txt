Kubernetes

Architecture -> master node,worker node (main work)

Container -> Running image

Pod-> Can have multiple container / manage multiple container

Service -> To explose pod outside world 

Deployment-> To manage the service we give replica counts and all

PVC (persistence volume claim) -> To store the data when any pod fails, Abstraction over the storage, PVC can be backed by NFS, Cloud storage AWS EBS

Stateful set -> manages state of the applications of state

Config Map -> Env var and all things

Secrets

Storage classes
